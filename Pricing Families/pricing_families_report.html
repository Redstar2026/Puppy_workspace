Observaciones y Supuestos del Análisis

Construcción de Familias: La llave de familia se construye con 5 campos: COST_UNIT + MARCA_DESC + N_MARCA + PAIS + N_FINELINE. Cada familia es única dentro del contexto (PAIS / FORMATO / ZONA). Se encontraron 121 familias, de las cuales 32 contienen más de un SKU y 89 son familias de un solo artículo.
SKUs sin VENTAS ni ROTACIONES (52): 52 artículos no tienen ventas registradas en SW 22. En estos casos, VENTAS = 0 para efectos del ranking. Si un grupo completo no tiene ventas, el PRIMARIO se elige por ITEM más bajo (criterio determinístico). Esto puede indicar artículos inactivos (ESTATUS = I), artículos sin surtido en tienda, o productos con venta real = 0.
Campo PRECIO_GPI_C_IMP: Es el precio de referencia para evaluar alineación. Este precio incluye impuestos (con IMP). Se usa como precio canónico de comparación dentro de cada familia.
Empates de ventas: En caso de empate en VENTAS, el desempate se realiza usando el número de ITEM más bajo (ascendente). Este criterio es 100% determinístico y reproducible en SQL (ROW_NUMBER() OVER(PARTITION BY ... ORDER BY VENTAS DESC, ITEM ASC)).
Dimensión única en el dataset: Todos los registros corresponden a PAIS=CR, FORMATO=SUPERMERCADOS, ZONA=SAN JOSE, TIENDA=4076. El análisis está preparado para escalar a múltiples países/formatos/zonas sin cambios en la lógica.
Rotación: La tabla contiene el campo ROTACIONES como única métrica de rotación disponible. Se incluye directamente como ROTACION en la base de salida. 52 artículos tienen ROTACION nula (coincide con los artículos sin VENTAS, son artículos inactivos o sin movimiento).
Categoría 894: El query incluyó CATEGORY_NBR IN (894, 6987). Los 162 registros encontrados pertenecen exclusivamente a la categoría 6987 (BEVERAGES - CARBONATED-D95). No se encontraron registros para la categoría 894 en esta tienda/semana/fecha.
Tolerancia de precio: Se considera "Igual al Primario" cuando la diferencia absoluta entre PRECIO_GPI_C_IMP del artículo y el del primario es menor a ₡0.01. Esto maneja posibles imprecisiones de punto flotante.

Esta lógica reproduce exactamente el análisis de familias y puede ejecutarse directamente en BigQuery. Diseñado para ser escalable a múltiples países, formatos y zonas.

Paso 1 — Base con Código de Familia

WITH base AS (
  SELECT
    -- Dimensiones de scope
    PAIS, FORMATO, NOMB_ZONE,
    -- Campos originales
    *,
    -- STEP 1: Llave de familia (5 campos) dentro del scope
    CONCAT(
      CAST(COD_PAIS    AS STRING), '_',
      CAST(COD_FORMATO AS STRING), '_',
      CAST(COD_ZONA   AS STRING),  '_',
      DENSE_RANK() OVER(
        PARTITION BY COD_PAIS, COD_FORMATO, COD_ZONA
        ORDER BY COST_UNIT, MARCA_DESC, N_MARCA, PAIS, N_FINELINE
      )
    ) AS CODIGO_FAMILIA_RAW
  FROM `wmt-k1-cons-data-users.k1_adhoc_tables.prcng_info_cam_a0g0dn_3_insumos_tienda`
  WHERE store_nbr = 4076
    AND EXTRACT_YR = 2026
    AND SW = 22
    AND EXTRACT_DATE = "2026-06-29 00:00:00 UTC"
    AND CATEGORY_NBR IN (894, 6987)
),

-- STEP 2: Identificar PRIMARIO (mayor ventas, tiebreak ITEM ASC)
ranked AS (
  SELECT
    *,
    ROW_NUMBER() OVER(
      PARTITION BY PAIS, FORMATO, NOMB_ZONE, CODIGO_FAMILIA_RAW
      ORDER BY
        COALESCE(VENTAS, 0) DESC,
        ITEM ASC     -- tiebreaker determinístico
    ) AS rn_primary
  FROM base
),

-- STEP 3: Extraer datos del PRIMARIO para hacer join
primarios AS (
  SELECT
    PAIS, FORMATO, NOMB_ZONE, CODIGO_FAMILIA_RAW,
    ITEM             AS ITEM_PRIMARIO,
    DESCRIPCION      AS DESC_PRIMARIO,
    PRECIO_GPI_C_IMP AS PRECIO_PRIMARIO,
    VENTAS           AS VENTAS_PRIMARIO,
    ROTACIONES       AS ROTACION_PRIMARIO
  FROM ranked
  WHERE rn_primary = 1
),

-- STEP 4: Join y calcular ESTADO_PRECIO
final AS (
  SELECT
    r.*,
    CONCAT('FAM_', r.CODIGO_FAMILIA_RAW) AS CODIGO_FAMILIA,
    CASE WHEN r.rn_primary = 1 THEN 'PRIMARIO' ELSE 'SECUNDARIO' END AS AGRUPACION,
    p.ITEM_PRIMARIO,
    p.DESC_PRIMARIO,
    p.PRECIO_PRIMARIO,
    p.VENTAS_PRIMARIO,
    p.ROTACION_PRIMARIO,
    CASE
      WHEN r.PRECIO_GPI_C_IMP IS NULL THEN 'Sin Precio'
      WHEN p.PRECIO_PRIMARIO  IS NULL THEN 'Sin Precio Primario'
      WHEN ABS(r.PRECIO_GPI_C_IMP - p.PRECIO_PRIMARIO) < 0.01
                                         THEN 'Igual al Primario'
      ELSE                                       'Diferente al Primario'
    END AS ESTADO_PRECIO,
    r.PRECIO_GPI_C_IMP - p.PRECIO_PRIMARIO AS DIFERENCIA_PRECIO,
    r.ROTACIONES AS ROTACION
  FROM ranked r
  LEFT JOIN primarios p
         USING (PAIS, FORMATO, NOMB_ZONE, CODIGO_FAMILIA_RAW)
)

-- ══ SALIDA 1: Base Detallada ══
SELECT * FROM final;


-- ══ SALIDA 2: Resumen Ejecutivo ══
SELECT
  PAIS, FORMATO, NOMB_ZONE AS ZONA,
  CODIGO_FAMILIA,
  MAX(MARCA_DESC)    AS MARCA,
  MAX(FINELINE_DESC) AS FINELINE,
  COUNT(*)           AS TOTAL_SKUS,
  ANY_VALUE(ITEM_PRIMARIO)    AS ITEM_PRIMARIO,
  ANY_VALUE(DESC_PRIMARIO)    AS DESC_PRIMARIO,
  ANY_VALUE(VENTAS_PRIMARIO)  AS VENTAS_PRIMARIO,
  ANY_VALUE(PRECIO_PRIMARIO)  AS PRECIO_PRIMARIO,
  COUNT(DISTINCT PRECIO_GPI_C_IMP)                                AS N_PRECIOS_DISTINTOS,
  COUNTIF(ESTADO_PRECIO = 'Igual al Primario')                   AS SKUS_IGUAL_PRECIO,
  COUNTIF(ESTADO_PRECIO = 'Diferente al Primario')               AS SKUS_DIFERENTE_PRECIO,
  COUNTIF(ESTADO_PRECIO IN ('Sin Precio', 'Sin Precio Primario')) AS SKUS_SIN_PRECIO,
  ROUND(
    COUNTIF(ESTADO_PRECIO = 'Igual al Primario') * 100.0 / COUNT(*),
    2
  ) AS PCT_ALINEACION
FROM final
GROUP BY 1,2,3,4
ORDER BY PCT_ALINEACION ASC;
